# Cclover Skills - 开发指南

## 项目简介

个人 AI Agent 技能集合，遵循 TDD 方法论开发，适用于任何智能体（OpenCode、Claude Code 等）。每个技能针对特定场景，帮助 Agent 更好地完成任务。

## 技术栈

- **技能格式**: Markdown (SKILL.md)
- **开发方法**: TDD (Test-Driven Development)
- **编写工具**: writing-skills 技能

## 项目结构

```
cclover-skills/
├── skills/              # 所有技能必须放在此目录
│   ├── skill-name/
│   │   ├── SKILL.md    # 技能主文件（必需）
│   │   └── *.*         # 支持文件（可选）
│   └── ...
├── AGENTS.md           # 本文件
└── README.md           # 项目说明
```

**重要规则**：所有技能必须放在 `skills/` 目录下，不能放在项目根目录。

## 开发规则

### 编写技能

**必须使用 writing-skills 技能编写**，除非用户明确表示不加载技能直接编写。

### 更新 README 文档

**添加或修改任何技能时，所有语言版本的 README 必须同步更新。**

必须更新的内容：
- 在 README.md 中更新技能列表（添加新技能或更新描述）
- 保持所有 README 语言版本中的技能描述一致
- 确保安装说明保持准确

这确保用户无论使用哪种语言都能发现和理解所有可用技能。
### TDD 测试流程

编写技能时必须进行 TDD 测试。**关键步骤**：

1. 使用 writing-skills 编写技能
2. 让子代理测试新技能时：
   - **直接告诉子代理刚编写的技能完整路径**
   - **让子代理全文读取技能文件**
   - 这一步非常重要，确保子代理能正确加载和理解技能

示例：
```
请全文阅读 /path/to/cclover-skills/skills/new-skill/SKILL.md，把它当成技能
```

### 技能设计原则

- **通用性**：技能应适用于所有 AI 助手，不依赖特定环境或工具
- **独立性**：技能可以独立使用
- **可组合性**：技能之间可以有相互依赖关系
