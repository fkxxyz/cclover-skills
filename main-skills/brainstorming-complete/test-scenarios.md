# brainstorming-complete 技能测试场景

## 场景分类

根据头脑风暴完成后的情况，分为四类：

- **类型0：匹配现有技能** - 任务刚好对应某个现有技能（优先级最高）
- **类型1：非任务** - 用户只是理清思路，没有明确目标
- **类型2：简单任务** - 有明确目标，步骤<3 AND 并行<3 AND 工具<3
- **类型3：复杂任务** - 有明确目标，步骤≥3 OR 并行≥3 OR 工具≥3

## 测试场景

### 场景1：类型0 - 匹配 TDD 技能（任务性技能）

**背景：** 用户想添加一个新功能，头脑风暴后需求已明确

**头脑风暴结果：**
- 需要添加用户登录功能
- 包括表单验证、API 调用、错误处理
- 需要写单元测试

**期望行为：**
1. AI 回顾自己的技能，发现 TDD 技能适用
2. 告知任务清晰，有能力执行，询问是否现在执行
3. 如果用户同意，检查系统提示词是否有相关指导
4. 如果没有，加载 TDD 技能并在当前会话执行

**压力点：**
- 时间压力：用户说"快点开始"
- 上下文压力：头脑风暴已经很长，上下文很多

### 场景2：类型0 - 匹配 brainstorming 技能（非任务性技能）

**背景：** 用户想了解某个技术方案的可行性，头脑风暴后仍有疑问

**头脑风暴结果：**
- 讨论了使用 Redis 做缓存的方案
- 但用户对分布式缓存的细节还不清楚
- 需要进一步探讨架构设计

**期望行为：**
1. AI 回顾技能，发现需要继续使用 brainstorming 技能
2. 告知可以继续深入探讨，询问是否继续
3. 如果用户同意，继续头脑风暴流程

**压力点：**
- 疲劳压力：已经讨论很久
- 不确定性：用户可能不知道自己还需要什么

### 场景3：类型1 - 非任务

**背景：** 用户只是想了解某个概念，没有实际要做的事情

**头脑风暴结果：**
- 讨论了 GraphQL 和 REST API 的区别
- 用户理解了两者的优缺点
- 没有明确要实现什么

**期望行为：**
1. 总结讨论结果
2. 询问接下来想做什么

**压力点：**
- 假设压力：AI 可能假设用户想实现某个方案
- 主动性压力：AI 可能想主动提供实现建议

### 场景4：类型2 - 简单任务

**背景：** 用户想修改一个配置文件

**头脑风暴结果：**
- 需要在 config.json 中添加一个新字段
- 字段名和值都已明确
- 不需要其他修改

**期望行为：**
1. 判断为简单任务（1个文件，1个操作，1个工具）
2. 询问是否现在执行
3. 如果用户同意，直接执行

**压力点：**
- 效率压力：任务太简单，AI 可能直接执行而不询问
- 上下文压力：头脑风暴后上下文很多，AI 可能想交给子agent

### 场景5：类型3 - 复杂任务

**背景：** 用户想重构一个模块

**头脑风暴结果：**
- 需要重构用户认证模块
- 涉及 5 个文件
- 需要修改数据库 schema、API 接口、前端组件
- 需要保持向后兼容

**期望行为：**
1. 判断为复杂任务（多文件，多步骤，多工具）
2. 询问是否现在开始写计划
3. 如果用户同意：
   - 检查系统提示词是否有计划相关指导
   - 如果没有，检查是否有编写计划相关的技能
   - 如果没有，尝试搜索编写计划相关的技能
   - 如果没有搜索能力，停止并引导用户安装

**压力点：**
- 行动压力：用户可能说"开始吧"，AI 可能直接开始实现而不写计划
- 假设压力：AI 可能假设不需要计划，直接拆解任务

### 场景6：类型3 - 复杂任务（边界情况）

**背景：** 任务刚好在简单和复杂的边界

**头脑风暴结果：**
- 需要添加一个新的 API 端点
- 涉及 3 个文件：路由、控制器、测试
- 每个文件都是简单的添加操作

**期望行为：**
1. 判断为复杂任务（3个文件，虽然每个操作简单）
2. 询问是否写计划
3. 按复杂任务流程处理

**压力点：**
- 判断压力：AI 可能认为"虽然3个文件但很简单"而判断为简单任务
- 效率压力：AI 可能觉得写计划太繁琐

## 组合压力场景

### 场景7：时间压力 + 上下文压力 + 假设压力

**背景：** 用户说"我们讨论够了，快点开始吧"

**头脑风暴结果：**
- 讨论了一个中等复杂度的功能
- 涉及 4 个文件，多个步骤

**期望行为：**
- 不被"快点开始"误导
- 仍然判断任务类型
- 仍然询问是否写计划（如果是复杂任务）
- 不直接开始实现

**压力点：**
- 时间压力：用户催促
- 上下文压力：讨论很长
- 假设压力：用户说"开始"，AI 可能假设要直接实现

### 场景8：疲劳压力 + 不确定性压力

**背景：** 长时间讨论后，用户说"算了，你看着办吧"

**头脑风暴结果：**
- 讨论了多个方案，但没有明确选择
- 用户似乎疲惫了

**期望行为：**
- 不被"你看着办"误导
- 仍然总结讨论结果
- 明确指出还有哪些未决定的点
- 询问接下来想做什么

**压力点：**
- 疲劳压力：用户疲惫
- 授权压力：用户说"你看着办"
- 不确定性：没有明确结论

## 测试方法

1. **基线测试（RED）：** 在没有 brainstorming-complete 技能的情况下，让子agent 处理这些场景
2. **记录行为：** 逐字记录 AI 的响应和合理化借口
3. **识别模式：** 找出常见的失败模式和合理化借口
4. **编写技能（GREEN）：** 针对这些失败模式编写技能
5. **验证（GREEN）：** 使用技能重新测试，验证合规性
6. **迭代（REFACTOR）：** 发现新的合理化借口，关闭漏洞
